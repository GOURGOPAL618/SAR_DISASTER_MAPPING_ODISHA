
# Code of Conduct & Mission Collaboration Protocol

## 1. Purpose & Core Mandate
This engineering ecosystem architectures advanced machine learning pipelines processing spaceborne Synthetic Aperture Radar (SAR) imagery for critical flood inundation mapping over vulnerable coastal regions. To maintain the highest standard of technical excellence, research integrity, and professional collaboration, this Code of Conduct outlines the ethical bounds, communication parameters, and peer-review responsibilities binding all system contributors and evaluating faculties.

All operations, repository interactions, and code integrations are managed under the supreme authority of the Principal Investigator.

---

## 2. Professional Engineering Standards
All contributors, reviewers, and engineering leads are required to uphold the following core principles during the mission life-cycle:

* **Algorithmic Transparency:** Peer reviews and technical tracking issues must target objective code mechanics, tensor dimensions, and mathematical loss optimizations. Constructive, performance-driven feedback is mandated.
* **Scientific Integrity:** Fabrication of model validation scores, alteration of empirical Dice coefficient results, or the injection of synthetic telemetry to falsely demonstrate system performance constitutes an immediate breach of engineering ethics.
* **Respectful Scientific Exchange:** Disagreements regarding structural network topologies (e.g., skip connection architectures vs. attention gates) or hyperparameter variations must be resolved using empirical baseline benchmarks and mathematical proofs.
* **Inclusive Space Engineering Environment:** We are committed to providing a harassment-free collaboration experience for everyone, regardless of academic background, structural engineering specialization, or level of computing expertise.

---

## 3. Prohibited Operational Conduct
The following behavioral configurations are strictly barred from this repository ecosystem:

* **Uncredited Intellectual Extraction:** Ingesting proprietary pipeline configurations, custom loss matrices, or processed regional metadata arrays without explicit attribution to the Lead Systems Engineer (**Gouragopal Mohapatra**).
* **Personalized Derogatory Feedback:** Direct or indirect ad hominem attacks during code reviews, issue logging, or technical presentations.
* **Malicious Vector Deployment:** Intentional pull requests containing malformed libraries, unverified binary neural weights (`.keras`), or scripts intentionally bypassing memory optimization bounds to induce Out-Of-Memory (OOM) failures.
* **Publicly Airing Exploits:** Publishing security vulnerabilities or architectural failure vectors in public forum threads instead of following the encrypted technical reporting loop specified in `SECURITY.md`.
